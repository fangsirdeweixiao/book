# StudyMate项目记忆与工作方法规则

## 🎯 核心项目记忆

### 1. 项目本质定位
```
这不是传统的小说创作项目
而是通过文化产品推广StudyMate的教育革命

核心目的：
├── 推广StudyMate平台，招募小b
├── 教育学生了解AI和协作能力  
└── 突出中国文化智慧价值
```

### 2. StudyMate产品记忆
- **网址：** www.studymate.cc
- **模式：** S2b2c社交电商
- **现状：** 产品已上线，需要规模化获客
- **瓶颈：** 你一个人承担所有角色

### 3. 核心教育理念
```
传统教育：知识记忆 → 考试检验 → 进入社会再学能力
AI时代教育：能力培养 → 实战检验 → AI辅助知识获取

五大核心能力：
├── 沟通能力（语言是载体）
├── 理解需求的能力（听懂别人要什么）
├── 团队协作的能力（与人共事）
├── 使用AI工具的能力（放大个体能力）
└── 跨文化理解能力（全球视野）
```

### 4. 任务驱动学习模式
```
不是"先学知识，再用知识"
而是"接到任务，边做边学"

有的放矢 → 学以致用 → 因人施教
```

## 🔧 工作方法规则

### 1. 创作内容规则
```
必须深度整合：
├── 道德经原文（每章对应章节）
├── StudyMate功能展示（自然植入）
├── AI技术节点（历史背景）
└── 中国文化智慧（哲学指导）

创作质量要求：
├── 文学价值：保持三体式宏大叙事
├── 思想深度：每章必须有哲学思考
├── 实用价值：服务于推广目标
└── 文化价值：突出东方智慧
```

### 2. 文件组织规则
```
/三体外篇/book/
├── 创作标准.md          # 核心规则文件
├── 第一部创作清单.md     # 37章创作计划
├── 内容创作/第一部/      # 章节内容
├── 资料库/              # 参考资料
├── Web界面/             # 可视化系统
└── JSON配置文件/         # 结构化数据
```

### 3. 协作流程规则
```
创作流程：
1. 我根据架构写初稿
2. 你审核指导修改方向
3. 我完善内容细节
4. 共同确定最终版本

审核标准：
├── 是否符合三个核心目的
├── 是否深度整合四大要素
├── 是否自然植入StudyMate
└── 是否达到文学质量标准
```

### 4. 质量保证规则
```
每章必须包含：
├── 道德经对应章节引用
├── StudyMate功能展示
├── AI技术背景介绍
├── 文化智慧体现
└── 推广价值传递

避免内容：
├── 过度营销感
├── 生硬植入
├── 文化偏见
└── 技术错误
```
